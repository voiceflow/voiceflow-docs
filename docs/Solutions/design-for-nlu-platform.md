## Intro to Connected Workflow

Voiceflow's Connected Workflow Solution allows for Voiceflow to be used as a collaborative, easy-to-use Conversation Design platform on-top of 3rd party NLU Platforms, such as Dialogflow.

### How it works

1. **(Optional) Import existing Assistant content into Voiceflow**
   - Use an Importer if available
2. **Non-technical teams create Assistant content in Voiceflow's Creator tool**
   - Design, test and iterate rapidly in Voiceflow's Creator tool
3. **Development teams pull Assistant content from Voiceflow using the Project API**
   - Use or build an [Adapter](<>)

### Problem it solves

As Assistants and Assistant teams get larger, it's common for NLU development platforms to become difficult to use, and collaborate in when building Assistants. This leads to design teams and non-technical stakeholders to rely on spreadsheets, word docs and other documents to create an Assistant's content.

Using the Connected Workflow solution, non-technical teams can create and update Assistant content in Voiceflow, maintaining a single source of truth for content, and decreasing the developer resources needed to make iterative changes to the Assistant.
